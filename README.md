# Pide — Product IDE

Prompts, tools, and plugins for product people who use AI coding assistants (Claude Code, Cursor, VS Code, and friends) as their everyday workspace.

This is a community collection of practical tips: small things that make the IDE feel like *your* IDE, not a developer's tool you happened to install.

## What's inside

Folders are added as content arrives — nothing empty, nothing speculative.

- [`tricks/`](./tricks) — copy-paste prompts and one-liners you hand to Claude Code (or your assistant of choice) to make the IDE behave the way you want

Future folders may include `tools/`, `plugins/`, `workflows/`, `settings/` — added when the first item lands.

## Contributing

1. Pick the folder that fits (or propose a new one in a PR).
2. Add a single markdown file. Use the format you see in existing tips:
   - A short summary in English (1–2 sentences)
   - The artifact itself (prompt, config, command) in a fenced code block, ready to copy
   - Platform / tool notes if relevant (macOS only, requires X extension, etc.)
3. Add a line for it in that folder's `README.md` index.
4. Open a PR. Keep it small — one tip per PR.

Tips can be in any language. If you write the artifact in your native language, a one-line English summary is enough to make it findable.
